## Disaster Recovery

### Backup Strategy

1. **Database Backups**
   - Automated daily snapshots (RDS)
   - Retention: 30 days
   - Point-in-time recovery enabled

2. **File Backups**
   - S3 versioning enabled
   - Cross-region replication
   - Lifecycle policies

3. **Configuration Backups**
   - Infrastructure as Code (Terraform)
   - Version controlled in Git

### Recovery Plan

| Scenario | RTO | RPO | Recovery Steps |
|----------|-----|-----|----------------|
| Database failure | 1 hour | 5 minutes | Restore from latest snapshot |
| Application crash | 5 minutes | 0 | Auto-scaling restarts containers |
| Region outage | 4 hours | 1 hour | Failover to backup region |
| Data corruption | 2 hours | 1 day | Restore from daily backup |
